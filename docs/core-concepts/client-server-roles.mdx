# Client Server Roles

Buyers and sellers share the x402 flow but have different jobs. The buyer proves willingness and capacity to pay; the seller protects resources and verifies guarantees.

## Why It Matters

4Mica is the clearing house for the agentic economy: a credit layer for x402 that lets agents and applications transact instantly, earn yield on collateral, and settle net positions later.

## How It Works

- Client handles the 402 response, policy checks, guarantee signing, and retry.
- Server advertises requirements, verifies and settles the guarantee, and serves content.
- Facilitator coordinates verification and guarantee issuance.
- Core protocol enforces collateral and settlement.

## Builder Guidance

Start with the simplest x402 flow that works, then add stronger policies, limits, validation, and monitoring as your payment surface grows. Keep buyer and seller responsibilities explicit so failures are easy to debug.

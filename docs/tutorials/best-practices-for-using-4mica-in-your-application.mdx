# Best Practices for Using 4Mica in Your Application

4Mica is payment infrastructure. Treat it with the same care you would give authentication, billing, and entitlement systems: clear policy, good logs, safe defaults, and predictable recovery paths.

## Start Narrow

- Use one network and one asset first.
- Protect one seller route or wrap one buyer client.
- Choose small test prices until retries and settlement are boring.
- Keep the first integration easy to reason about before adding validation gates or multiple networks.

## Build With Policy

- Set max spend per request and per task.
- Allowlist seller domains for autonomous agents.
- Require explicit approval for new networks or assets.
- Keep signing keys in secure runtimes or managed wallet infrastructure.

## Operate The Flow

- Log guarantee ID, request ID, payer, recipient, asset, network, and settle status.
- Alert on repeated verification failures, insufficient collateral, missed settlement deadlines, and defaults.
- Make withdrawal flows aware of open guarantees and settlement obligations.
- Document what support should do when a guarantee is pending, payable, netted, settled, disputed, cancelled, or defaulted.

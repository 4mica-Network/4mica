# Building a Pay-Per-Use API with 4Mica

In this walkthrough, you will turn a normal API endpoint into a route that agents can pay for on credit. The seller keeps its HTTP interface, the buyer pays through x402, and 4Mica handles guarantees, clearing, and delayed settlement.

## What You Will Build

- A protected API route with a price per request.
- A buyer request that receives HTTP 402, signs a guarantee, and retries automatically.
- Basic logging for guarantee ID, request ID, payer, and settlement result.

## Steps

1. Create a normal Express or FastAPI endpoint.
2. Install the 4Mica x402 middleware for your framework.
3. Configure scheme, price, network, payTo address, and guarantee version.
4. Run a funded buyer client against the route on Base Sepolia.
5. Verify that the paid response only returns after payment validation succeeds.

## Finished Shape

By the end, your API should feel like any other HTTP service to the buyer. The difference is that unpaid requests receive machine-readable payment requirements, and paid retries arrive with a collateral-backed guarantee that your server can verify and settle.

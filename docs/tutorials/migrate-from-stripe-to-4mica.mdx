# Migrate from Stripe to 4Mica

Stripe is excellent for human checkout, subscriptions, and invoices. 4Mica is designed for machine-speed usage where agents pay per request and should not create accounts, manage prepaid credits, or wait for card-based billing flows.

## What To Move First

- Metered API calls where the buyer is an agent or automated system.
- Usage that is currently prepaid into isolated balances.
- Low-value high-frequency work where on-chain settlement per call is too expensive.
- Internal agent workflows where you control both buyer and seller during the migration.

## Migration Plan

1. Keep Stripe for existing human plans while introducing 4Mica for agent usage.
2. Map each automated customer or agent to a wallet address and payment policy.
3. Replace prepaid balances with collateral deposits and tab-based credit.
4. Run both systems in parallel and compare usage, settlement, and support volume.
5. Move more traffic once failed payments, refunds, and claims have clear operational paths.

## What Changes For Users

Users do not need to manage per-service balances. They fund collateral, let agents spend against it, and settle net obligations later. Sellers still get a verifiable payment path before serving paid work.

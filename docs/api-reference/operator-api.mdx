# Operator API

The operator API exposes core network state used by SDKs, dashboards, and operational services. Most product integrations should prefer the SDKs unless they need direct system visibility.

## Common Endpoints

- POST /auth/nonce: create a SIWE nonce and message template.
- POST /auth/verify: verify a wallet signature and issue tokens.
- GET /core/health: check database and chain RPC dependencies.
- GET /core/public-params: return operator public key, contract address, accepted guarantee versions, and validation registry parameters.
- GET /core/tokens: list supported ERC-20 tokens for the active deployment.
- POST /core/guarantees: issue a BLS guarantee for a signed request.

## Authentication

Use SIWE for user-authenticated flows. Server-to-server usage should keep bearer tokens in backend environments only and rotate them according to your operational policy.

## When To Use It

Use the operator API for dashboards, internal reconciliation, supported-token discovery, and advanced protocol operations. Use seller middleware for ordinary protected routes and buyer wrappers for ordinary paid requests.

# Frontend

Frontend usage depends on where the signer lives. Browser apps can guide users through wallet connection and registration, while agents and server-side clients usually run the automatic paid-request wrapper in a trusted runtime.

## User Interface Responsibilities

- Explain the network, asset, and amount before deposit.
- Show when deposits are waiting for chain finalization.
- Display pending guarantees and settlement state when available.
- Avoid asking users to sign opaque payment policies.

## Agent Client Responsibilities

- Use a secure signer or managed wallet.
- Wrap HTTP calls with x402 payment handling.
- Apply policy before signing a payment guarantee.
- Surface failed 402 retries as actionable errors.

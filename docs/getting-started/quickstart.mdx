Get started with 4Mica by registering collateral, wiring a buyer client, and making your first credit-backed x402 request.

After you complete this guide, you will have a wallet ready to pay with 4Mica credit and a client that can handle HTTP 402 responses automatically.

## Before you begin

4Mica adds a credit layer to x402. Instead of sending an on-chain transaction for every paid request, a payer deposits collateral once, signs payment guarantees as requests happen, and settles the tab later.

You need:

- A wallet private key or supported signer for your agent or service.
- Testnet ETH for gas if you are using Base Sepolia or Ethereum Sepolia.
- Collateral in ETH, USDC, or USDT, depending on the network and route.
- Node.js if you want to follow the example below.

Use Base Sepolia (`eip155:84532`) for development. Move to Base (`eip155:8453`) when you are ready for production traffic.

## Register collateral

Deposit collateral into 4Mica before your first paid request. This registers the wallet and gives 4Mica a balance to secure future payment guarantees.

During registration, choose the same network you plan to use in your x402 client. Wait for the deposit to finalize before continuing.

## Install the packages

Install the 4Mica x402 client, the x402 fetch wrapper, and viem:

```bash
pnpm install @4mica/x402 @x402/fetch viem
```

## Make your first paid request

Wrap your normal `fetch` client with the 4Mica scheme. When the API returns HTTP 402 payment requirements, the wrapper opens or reuses a tab, signs the guarantee, retries the request, and returns the paid response.

```ts
import { wrapFetchWithPaymentFromConfig } from "@x402/fetch";
import { FourMicaEvmScheme } from "@4mica/x402/client";
import { privateKeyToAccount } from "viem/accounts";

const account = privateKeyToAccount(process.env.PRIVATE_KEY);
const scheme = await FourMicaEvmScheme.create(account);

const fetchWithPayment = wrapFetchWithPaymentFromConfig(fetch, {
  schemes: [{ network: "eip155:84532", client: scheme }],
});

const response = await fetchWithPayment("https://api.example.com/data");
const data = await response.json();
```

If the request succeeds, your client received the protected response and the seller received a settled payment guarantee.

## Protect a route

If you are building the seller side, protect one route with the 4Mica payment middleware. Start with a small test price and a seller-hosted tab endpoint.

```ts
app.use(paymentMiddlewareFromConfig({
  "GET /premium-content": {
    accepts: {
      scheme: "4mica-credit",
      price: "$0.10",
      network: "eip155:84532",
      payTo: "0xYourAddress",
    },
  },
}, { advertisedEndpoint: "https://api.example.com/payment/tab" }));
```

The middleware returns HTTP 402 responses, advertises the tab endpoint, verifies payment payloads, settles guarantees, and serves the protected route after payment succeeds.

## Confirm it worked

A successful first run should end with:

- The buyer receiving a normal HTTP response from the protected route.
- The seller receiving a settled guarantee for the request amount.
- A tab that can be paid later by the buyer or claimed by the seller if the buyer defaults during the claim window.
